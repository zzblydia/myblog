# TelegramGroupMessageAssistant
这是一个 telegram group message assistant 项目, 功能是提供群消息方面的辅助, 而非管理群组功能.
当使用者较忙或群消息过多而无法查看每一条群消息时, 由本程序对群消息中有价值的信息进行提取和汇总.

## 架构方案
1.代码实现基于python 3.11.2.
2.运行模式是telegram bot时,使用的开源方案为`python-telegram-bot`(v22.7)
3.运行模式是telegram client时,使用的开源方案为`Telethon`

## 主要功能说明
1.程序的功能是监听指定群组的消息,并进行本地记录和汇总.
2.程序运行模式可以是telegram bot,也可以是telegram client(因为不是所有群组都允许bot进入).
3.程序可以调用大模型对聊天记录进行解析和汇总.

## 详细功能设计

### 配置文件
1.优先读取配置文件`config_local.json`,如不存在则读取`config.json`.
2.配置文件格式如下:
```
{
    "bot_token" : "abc",
    "run_mode" : "bot",
    "watch_group_list" : ["name1", "name2"],
    "owner_id" : "abc123",
    "notify_group" : "abcd"
}
```

### 运行模式
#### telegram client模式
##### 群组功能
1.用户加入群组后,如果群组在关注列表里(配置中的`watch_group_list`或列表为空),则开始接收群组消息(主要是文字和图片消息, 音频和视频消息暂不处理).
2.每个小时分析群聊天记录, 如果发现有价值的聊天信息, 则在群组(配置中的`notify_group`)发送消息, 如发现不在通知群组, 则仅打印日志.

#### telegram bot模式
##### 群组功能
1.机器人被拉入群组后,如果群组在关注列表里(配置中的`watch_group_list`或列表为空),则开始接收群组消息(主要是文字和图片消息, 音频和视频消息暂不处理).
2.机器人被拉入群组后,仅当机器人是群组管理员时,才会每个小时在群组发送消息, 消息内容为群组当天的聊天信息汇总.消息最后提供按键, 支持查询过去1周, 过去1个月, 今年的聊天记录分析结果.
3.机器人被拉入群组后,如果机器人

##### 私聊功能(仅限bot运行模式)
1.用户私聊机器人，可指定目标群组和时间范围（昨日、一周、一月、三个月）申请群聊总结。
2.用户发送查询命令后, 查询和用户都有哪些公共群组, 如果和用户没有公共群主, 则提示用户. 如果有公共群组, 则给用户一个群名列表(不超过10个),用户点击后, 返回群组聊天信息提取成果给用户.

### 聊天记录
1.有价值的聊天信息是指统计聊天消息中的高频词汇和高频话题,并进行汇总,汇总形式比如top10关键词..
2.针对聊天消息中的新话题, 如符合群组标题或群组聊天氛围, 则在汇总时单独罗列.
