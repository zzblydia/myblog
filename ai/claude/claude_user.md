## 编码要求
1.在开始编码前,请和我讨论清楚方案的目的和实现细节,禁止直接开始编码.
2.在设计编码方案时,请优先考虑方案的可行性:
 - 方案引用的开源组件必须事实存在
 - 方案不存在开源仓库issue里已知的问题
 - 方案具备可测试性
 3.关键代码必须有准确的注释,注释量不少于总代码量的30%.请保持注释的简洁.

### c language
1.请基于C98 C11进行编码,确保语法兼容.

### python
1.请基于Python 3.11.2编码,确保语法兼容.
2.在代码简洁的基础上优先使用官方的库.
3.代码具备单文件调试功能(__main__).
4.代码开始执行时,总是先初始化日志模块,日志文件命名默认是时间戳, 默认级别是info,同时输出到文件和控制台,对于日志的级别要设置恰当,以确保日志运行不会太多日志打印.

### java
1.请基于java 8进行编码,确保语法兼容.
